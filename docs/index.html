Overview

The 10-second explanation

Most ad systems are brittle because they match strings. LLM apps need meaning-based matching.

SponsorStream-MCP is a lightweight semantic ad engine: your LLM app requests relevant sponsor inserts using natural language context, and SponsorStream-MCP matches ads semantically using embeddings + vector search.

Architecture

Most "monetization plugins" become a security disaster because runtime can mutate data. SponsorStream-MCP splits responsibilities into a Dual-plane architecture.

Data Plane (Runtime)

Ad matching, read-only retrieval. Called by LLMs.

uv run ad-mcp-data
  • ads_match: Semantic matching with constraints
  • ads_explain: Audit trace
  • ads_health: Liveness/readiness

Control Plane (Admin)

Provisioning, ingestion, ops. Called by Humans/CI.

uv run ad-mcp-control
  • collection_ensure: Create collection
  • ads_upsert_batch: Ingestion
  • ads_delete: Cleanup

Get Started

Prerequisites

Python 3.10+, uv, Docker (for Qdrant)

1. Install uv

curl -LsSf https://astral.sh/uv/install.sh | sh

2. Start Qdrant

docker run -d --name qdrant -p 6333:6333 -p 6334:6334 qdrant/qdrant

3. Sync & Seed

uv sync
uv run ad-index create
uv run ad-index seed

4. Run Servers

# Data Plane
uv run ad-mcp-data

# Control Plane
uv run ad-mcp-control

Technical Deep Dive

SponsorStream-MCP is built on local embeddings (FastEmbed) and Qdrant. Context embedding stays local; storage and retrieval use your own infrastructure.
