Engine / Studio Architecture

Separation of concerns for security and reliability.

Engine (Runtime)

Read-Only

Handles campaign matching and retrieval. Optimized for low latency. Called by LLMs.

uv run sponsorstream-engine
  • Semantic matching
  • Audit tracing
  • Health checks

Studio (Admin)

Write Access

Manages provisioning, ingestion, and operations. Called by Humans or CI pipelines.

uv run sponsorstream-studio
  • Collection management
  • Batch ingestion
  • Analytics reporting

Ready to Stream?

Get up and running in minutes with Python 3.10+, uv, and Docker.

1. Install uv 
curl -LsSf https://astral.sh/uv/install.sh | sh
2. Start Qdrant 
docker run -d --name qdrant -p 6333:6333 -p 6334:6334 qdrant/qdrant
3. Sync & Seed 
uv sync && uv run sponsorstream-cli create && uv run sponsorstream-cli seed
4. Run Servers 
uv run sponsorstream-engine
uv run sponsorstream-studio
